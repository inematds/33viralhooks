De prompt avulso a sistema (sua Hook Machine)

Prompt avulso resolve um vídeo. Uma skill (sistema reutilizável) resolve os próximos 100: você dá o nicho, ela devolve hooks + camadas + prompts de imagem, sempre no mesmo padrão.

🆕 Novo aqui? O que é uma "skill"

No Claude Code (e ferramentas similares), uma skill é uma pasta com um arquivo SKILL.md que descreve QUANDO usá-la e O QUE fazer. Você invoca por um comando e ela roda o processo inteiro — como ter um roteirista de plantão com seu método embutido.

🧠 O que sua Hook Machine entrega

null

  • •Recebe: um nicho/assunto e o idioma.
  • •Gera: 20-33 hooks na fórmula, agrupados por padrão.
  • •Completa: as 3 camadas de cada hook.
  • •Produz: o prompt de imagem de cada Visual Hook.
  • •Filtra: pontua com a rubrica e corta os fracos.
2

Anatomia de uma SKILL.md

Uma skill é simples: um cabeçalho que diz QUANDO ativar e um corpo que diz COMO agir. Vamos ver as partes antes de colar a sua.

🧠 As partes de uma SKILL.md

null

  • •name: nome curto da skill (ex.: hook-machine).
  • •description: quando usar — os gatilhos ("criar hooks", "gerar ganchos virais").
  • •Corpo (instruções): o passo a passo que a IA segue — papel, fórmula, formato de saída, rubrica.

💡 Regra da boa description

A description é o que faz a skill "acender" na hora certa. Liste gatilhos reais que você diria: "gera hooks pra…", "ganchos virais de…", "ideias de Reels sobre…".

# Hook Machine

Você é roteirista de vídeo curto especialista em hooks virais.

## Quando o usuário der um NICHO/ASSUNTO e (opcional) idioma:

1. Gere 20 hooks (ou a quantidade pedida) na fórmula:
   - Context Lean: começa no pico, zero contexto inútil.
   - Scroll Stop: a cena/afirmação que trava o dedo nos 3s.
   - Contrarian Snapback: contraria uma crença comum do nicho.
2. Distribua entre os 5 padrões: medo, contrarian, número, prova pessoal, curiosidade.
3. Para cada hook entregue as 3 CAMADAS:
   - Frase falada (máx 12 palavras)
   - Visual Hook (cena concreta, legível no mudo)
   - Text Hook (2-3 palavras)
4. Pontue cada hook com a rubrica (0/1): específico, lacuna, contrarian, visual, curto. Mostre só os com nota >= 4.
5. Para os aprovados, gere o PROMPT DE IMAGEM (inglês, cinematográfico, terminando em "no text, no watermark").

## Regras
- Específico vence genérico (números, valores, prazos).
- PROIBIDO clichê, abertura genérica, promessa vaga.
- Saída final em tabela: # | Frase falada | Visual Hook | Text Hook | Padrão | Nota | Prompt de imagem.

## Idioma
Responda no idioma pedido (default: português do Brasil).
✅ Como verificar: No Claude Code: salve em hook-machine/SKILL.md e peça "gera hooks de ". Em outro chat: cole o corpo como primeira mensagem e mande o nicho. Deve vir a tabela com nota e prompt de imagem.

Pipeline em lote: do nicho ao pacote pronto

Com a skill no lugar, dá pra rodar em lote — vários nichos, vários hooks, imagens incluídas. Aqui o passo a passo reproduzível.

🧠 O pipeline, passo a passo

null

  • •1. Gerar: rode a Hook Machine pro nicho → tabela de hooks aprovados.
  • •2. Imagens: pegue a coluna "prompt de imagem" e gere cada uma no flux/Midjourney.
  • •3. Montar: junte frase + visual + texto + imagem num card (igual aos da Trilha 2).
  • •4. Agendar: distribua os hooks em vídeos ao longo da semana.
🎯 Objetivo: Gerar todas as imagens de um lote a partir de uma planilha de prompts (Node + servidor local).
// salve como gen.js — lê prompts.json [{n, prompt}] e salva PNGs
const http=require('http'),fs=require('fs');
const items=JSON.parse(fs.readFileSync('prompts.json','utf8'));
(async()=>{ for(const it of items){
  const payload=JSON.stringify({model:'flux2-klein',prompt:it.prompt,width:1024,height:576,steps:4,guidance_scale:1.0,seed:it.n});
  const png=await new Promise((ok,no)=>{const r=http.request({host:'localhost',port:8000,path:'/generate',method:'POST',headers:{'Content-Type':'application/json','Content-Length':Buffer.byteLength(payload)}},res=>{let b='';res.on('data',c=>b+=c);res.on('end',()=>ok(Buffer.from(JSON.parse(b).image,'base64')))});r.on('error',no);r.write(payload);r.end();});
  fs.writeFileSync('hook-'+it.n+'.png',png); console.log('ok',it.n);
}})();
✅ Como verificar: Tenha o inemaimg rodando em localhost:8000. Rode "node gen.js": deve criar hook-1.png, hook-2.png… Conte os arquivos == número de prompts.
5

Banco de hooks: reusar, taguear, versionar

Hook bom não é descartável. Guarde os aprovados num banco — uma planilha ou JSON — pra reusar, remixar e ver o que repete sucesso.

🧠 Estrutura mínima do banco

null

  • •id, nicho, padrão — pra filtrar depois.
  • •3 camadas — frase, visual, texto.
  • •status — ideia / gravado / publicado.
  • •desempenho — views/retenção quando publicar.
🎯 Objetivo: Padronizar uma entrada do seu banco de hooks (JSON reutilizável).
{
  "id": "fin-001",
  "nicho": "financas",
  "padrao": "numero",
  "frase_falada": "Transformei US$500 em US$15.000 sem tocar na bolsa.",
  "visual_hook": "nota de 500 se multiplicando em maços de dinheiro",
  "text_hook": "No Stocks",
  "status": "ideia",
  "views": null
}
✅ Como verificar: Crie banco.json como uma lista desses objetos. Pra reusar: filtre por "padrao" ou "nicho" e adapte. Quando publicar, preencha "views" e veja qual padrão rende mais pra você.

💡 Versionar com Git

Guarde o banco.json num repositório (você aprende isso no curso "Do Zero ao Deploy" do INEMA). Cada vídeo publicado vira um commit — histórico do que funcionou.

6

Publicar e medir: o ciclo que faz você melhorar

O hook só prova seu valor no campo. Feche o ciclo: teste, meça, dobre no que funciona. É assim que você sai de "copiando hooks" pra "tendo um método".

🧠 O ciclo de melhoria

null

  • •Teste: grave 2-3 versões do mesmo conteúdo com hooks diferentes.
  • •Meça: olhe retenção nos 3s e % que passou de 50% — não só views.
  • •Aprenda: qual PADRÃO segurou mais? (medo? número?)
  • •Dobre: repita o padrão vencedor no seu nicho.

⚠️ Não confie em 1 vídeo

Um hook pode viralizar por sorte e outro morrer por horário. Olhe a média de 5-10 vídeos por padrão antes de concluir o que funciona pra você.
